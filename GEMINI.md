# Gemini Project Configuration

This file stores key configuration details and conventions for the "The Amy Project" to ensure consistent and efficient assistance.

## Architectural Rules
*   **Prioritize Framework Standards**: Always research the "ADK way" before implementing custom solutions. If a framework interface exists (e.g., `SessionService`), implement it rather than bypassing it.
*   **Whole Context Awareness**: Before modifying core logic, trace the entire data flow (from input to persistence) to ensure no duplication or conflicts exist with framework internals.
*   **Avoid "Sidecar" Logic**: Do not build parallel systems that duplicate framework responsibilities (e.g., manual memory management alongside ADK runners).
*   **Depend on Tools**: Use Agent Tools for all external side effects (memory, searching, etc.) rather than hardcoding logic in the runtime loop.

## Project Structure

*   **`amy/`**: Main application source code.
    *   **`core/agent.py`**: Defines the primary `root_agent` and factory.
    *   **`core/amy.py`**: Main application entry point (`Amy` class).
    *   **`integrations/`**: specific platform integrations (Telegram, Web).
*   **`scripts/`**: Utility scripts and runners.
*   **`venv/`**: Python virtual environment.
*   **`.env`**: Stores environment variables (API keys, tokens).
*   **`requirements.txt`**: Lists all Python dependencies.
*   **`start_telegram.sh`**: Launcher script for the Telegram bot.
*   **`start_web.sh`**: Launcher script for the ADK web UI.

## Key Files & Variables

*   **Agent Definition**: `amy/core/agent.py` contains the `root_agent`.
*   **Database Location**: The SQLite database is located at `instance/amy_memory.db`.
*   **Environment File**: The `.env` file is located in the project root.
